# Contributing to Pitchfork Pro

از مشارکت شما در پروژه Pitchfork Pro استقبال می‌کنیم! این راهنما به شما کمک می‌کند تا در پروژه مشارکت کنید.

## 🎯 چگونه می‌توانید مشارکت کنید؟

### 1. گزارش باگ
اگر باگی پیدا کردید:
- در بخش Issues گیت‌هاب یک issue جدید ایجاد کنید
- عنوان واضح و مختصر بنویسید
- مراحل بازتولید باگ را توضیح دهید
- نسخه MetaTrader و سیستم عامل خود را ذکر کنید

### 2. پیشنهاد ویژگی جدید
برای پیشنهاد ویژگی جدید:
- در بخش Issues درخواست خود را ثبت کنید
- کاربرد ویژگی را توضیح دهید
- مثال‌هایی از نحوه استفاده ارائه دهید

### 3. ارسال کد (Pull Request)
مراحل ارسال کد:

#### الف) Fork کردن پروژه
```bash
# روی دکمه Fork در گیت‌هاب کلیک کنید
```

#### ب) Clone کردن
```bash
git clone https://github.com/YOUR_USERNAME/PitchforkPro.git
cd PitchforkPro
```

#### ج) ایجاد Branch جدید
```bash
git checkout -b feature/YourFeatureName
# یا برای رفع باگ
git checkout -b bugfix/YourBugFix
```

#### د) انجام تغییرات
- کد خود را بنویسید
- از استانداردهای کدنویسی MQL5 پیروی کنید
- کامنت‌های لازم را اضافه کنید

#### ه) Commit کردن
```bash
git add .
git commit -m "Add: description of your changes"
```

#### و) Push کردن
```bash
git push origin feature/YourFeatureName
```

#### ز) ایجاد Pull Request
- به صفحه اصلی ریپازیتوری بروید
- روی دکمه "Compare & pull request" کلیک کنید
- توضیحات کاملی از تغییرات بنویسید
- PR را ارسال کنید

## 📝 استانداردهای کدنویسی

### نام‌گذاری
- کلاس‌ها: `PFP_ClassName` (PascalCase)
- توابع: `FunctionName` (PascalCase)
- متغیرها: `variableName` (camelCase)
- ثابت‌ها: `CONSTANT_NAME` (UPPER_CASE)

### ساختار فایل
```mql5
//+------------------------------------------------------------------+
//|                                              FileName.mqh        |
//|                                    Copyright 2024, Your Name     |
//|                                             https://yoururl.com  |
//+------------------------------------------------------------------+
#property copyright "Copyright 2024, Your Name"
#property link      "https://yoururl.com"
#property version   "1.00"
//+------------------------------------------------------------------+

#include <OtherFile.mqh>

// کلاس‌ها و توابع
//+------------------------------------------------------------------+
```

### کامنت‌گذاری
- برای هر کلاس و تابع مهم کامنت بگذارید
- پارامترها و مقدار بازگشتی را توضیح دهید
- از کامنت‌های خطی برای توضیح منطق پیچیده استفاده کنید

## 🧪 تست کردن

قبل از ارسال PR:
1. کد را در MetaEditor کامپایل کنید
2. بدون خطا بودن کامپایل را بررسی کنید
3. روی چارت تست کنید
4. عملکرد صحیح را تأیید کنید

## 📋 بررسی PR

PR شما باید:
- [ ] بدون خطا کامپایل شود
- [ ] با استانداردهای کدنویسی مطابقت داشته باشد
- [ ] کامنت‌های لازم را داشته باشد
- [ ] تست شده باشد
- [ ] توضیحات کافی در مورد تغییرات داشته باشد

## 💬 ارتباطات

- برای سوالات عمومی از Discussions استفاده کنید
- برای باگ‌ها و ویژگی‌ها از Issues استفاده کنید
- برای تغییرات کد از Pull Requests استفاده کنید

## 📄 مجوز

با ارسال کد، شما موافقت می‌کنید که کد شما تحت مجوز MIT منتشر شود.
